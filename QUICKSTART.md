# 快速启动指南

## 第一步：安装依赖

```bash
yarn install
```

## 第二步：启动开发服务器

```bash
yarn serve
```

或者：

```bash
yarn dev
```

项目将在浏览器中自动打开 `http://localhost:3000`

## 第三步：配置后端 API（可选）

如果需要连接后端 Spring Boot API，请创建 `.env` 文件：

```env
VITE_API_BASE_URL=http://localhost:8080/api
```

如果不配置，项目会使用模拟数据运行。

## 功能说明

### 已实现的功能

1. **首页** (`/home`)
   - 数据统计展示
   - 热门药材和药膳推荐

2. **药材库** (`/herbs`)
   - 药材列表展示
   - 按性味、归经筛选
   - 关键词搜索
   - 分页功能

3. **药材详情** (`/herbs/:id`)
   - 详细信息展示
   - 收藏功能
   - 评论功能

4. **药膳配方** (`/recipes`)
   - 配方列表展示
   - 关键词搜索
   - 分页功能

5. **配方详情** (`/recipes/:id`)
   - 详细制作步骤
   - 关联药材跳转
   - 收藏和评论功能

6. **常见病症** (`/diseases`)
   - 病症列表展示
   - 关键词搜索
   - 分页功能

7. **病症详情** (`/diseases/:id`)
   - 病因病机说明
   - 防治方法
   - 关联药材和药膳
   - 收藏和评论功能

## 组件说明

### ContentCard 组件
通用内容卡片组件，支持：
- 图片展示（带占位符）
- 标题和描述
- 标签展示
- 收藏功能

### CommentSection 组件
评论组件，支持：
- 发表评论
- 评论列表展示
- 时间格式化显示

### CategoryFilter 组件
筛选组件，支持：
- 关键词搜索
- 性味筛选（药材）
- 归经筛选（药材）
- 重置功能

## 注意事项

1. 项目使用 Vue 3 Composition API (`<script setup>`)
2. 所有 API 调用都有错误处理，失败时会使用模拟数据
3. 收藏功能目前使用本地状态管理，需要后端支持持久化
4. 响应式设计，支持移动端和桌面端

## 常见问题

**Q: 启动后页面空白？**
A: 检查浏览器控制台是否有错误，确保所有依赖已正确安装。

**Q: API 请求失败？**
A: 项目已配置错误处理，API 失败时会自动使用模拟数据，不影响开发。

**Q: 如何修改端口？**
A: 修改 `vite.config.ts` 中的 `server.port` 配置。
